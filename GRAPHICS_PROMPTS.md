## üìê Technical Specifications

### General Guidelines:
- **Format:** PNG with alpha channel (transparency)
- **Color Mode:** RGBA (32-bit)
- **Pixel Style:** Pixel art or clean cartoon style
- **Anti-aliasing:** Minimal or none (for pixel art look)
- **Padding:** Include 1-2 pixel padding around sprites to prevent edge bleeding

### Color Palette Suggestions:
- **Snake P1:** #4CAF50 (green), #2E7D32 (dark green), #81C784 (light green)
- **Snake P2:** #FF9800 (orange), #E65100 (dark orange), #FFB74D (light orange)
- **Frog Small:** #4CAF50 (green), #1B5E20 (dark green)
- **Frog Medium:** #2196F3 (blue), #0D47A1 (dark blue)
- **Frog Large:** #FF5722 (red-orange), #BF360C (dark red)
- **Water:** #4FC3F7 (cyan), #0288D1 (dark cyan)
- **Effects:** #FFD700 (gold), #FFF59D (light yellow)

### Animation Notes:
If creating sprite sheets for animation:
- Arrange frames horizontally (left to right)
- Keep frame size consistent
- Include 2-4 frames for simple animations
- Label with frame count (e.g., `snake_walking_4frames.png`)

## üéØ Usage Instructions

1. Generate each asset using the prompts above
2. Save with the exact filename specified
3. Create a `/assets/` folder in the game directory
4. Organize into subfolders:
   - `/assets/characters/` - Snake sprites
   - `/assets/enemies/` - Frog sprites
   - `/assets/environment/` - Lily pads, backgrounds
   - `/assets/effects/` - Particle and effect sprites
   - `/assets/ui/` - Interface elements

5. Update `game.js` to load these images instead of drawing colored rectangles
